# Mod authoring guide

## Folder layout

```
NOLoader/mods/MyMod/
  mod.json
  MyMod.dll
  patch.bake.json    # RDYTU hash-only (generated by pack-mod-rdytu)
```

One folder per mod. Dependencies listed in `mod.json`.

---

## `mod.json` (DEV.SDK)

```json
{
  "id": "com.example.mymod",
  "guid": "00000000-0000-0000-0000-000000000001",
  "version": "1.0.0",
  "name": "My Mod",
  "assembly": "MyMod.dll",
  "entryType": "MyMod.MyModEntry",
  "loadStage": "MainMenu",
  "dependencies": [],
  "patches": [
    {
      "target": "SomeGameType::SomeMethod",
      "inject": "MyMod.Patches::SomePrefix",
      "method": "Prefix",
      "expectedSignatureHash": "ABCDEF0123456789"
    }
  ]
}
```

### Fields

| Field | Description |
|-------|-------------|
| `id` | Unique string id (DEV) |
| `guid` | Optional stable GUID |
| `version` | Semver string for display |
| `assembly` | DLL filename in same folder |
| `entryType` | Fully qualified type implementing `INOMod` |
| `loadStage` | `PreMenu`, `MainMenu`, or `Mission` |
| `dependencies` | List of other mod `id` values |
| `patches` | Cecil patches applied to game assemblies |

### Patch target syntax

`Namespace.Type::Method` or `Type::Method` for types in `Assembly-CSharp`.

Game types must match the API dump — verify in `_Nuclear_Option_\Assembly-CSharp\`.

### Patch inject syntax

`YourAssembly.YourType::MethodName` — static method, signature must match patch kind (Prefix/Postfix/PrefixSkip).

## `INOMod` entry point

```csharp
using NOLoader.API;

public sealed class MyModEntry : INOMod
{
    public void OnLoad(ref NOModContext ctx)
    {
        // ctx.GameRoot, ctx.ModRoot, ctx.ModId, ctx.Stage
    }

    public void OnUnload(ref NOModContext ctx) { }
}
```

Reference `NOLoader.API.dll` from `NOLoader/core/` or project reference in DEV.SDK.

## RDYTU hash-only manifests

Player releases strip plain `id` and patch strings from shipped `mod.json`:

- `idHash` — Murmur32 of mod id
- `targetHash` / `injectHash` — Murmur32 of patch descriptors
- `patch.bake.json` — maps hashes → plain strings for PatchTool at pack time

Pack:

```powershell
.\scripts\pack-mod-rdytu.ps1 -ModFolder ... -OutputFolder ...
```

---

## IL patch stages

Patches in `mod.json` apply when bootstrap reaches that stage:

| Mod loadStage | Patches applied |
|---------------|-----------------|
| PreMenu | At bootstrap |
| MainMenu | With main menu activation |
| Mission | When mission scene loads |

Core bootstrap patches (MainMenu hook, etc.) are separate — applied by PatchTool.

---

## Registry mods

Use `NOModRegistry` API (`NOLoader.Registry`) to register missiles/weapons/aircraft by hash key. See `DEV.SDK/mods/NOLoader.RegistrySample/`.

Encyclopedia inject is automatic via core Cecil hook on `Encyclopedia::AfterLoad`.

---

## Best practices

1. Always ship `expectedSignatureHash` for every patch (Gate L2).
2. Prefer `MainMenu` load stage unless you need PreMenu/Mission.
3. Avoid per-frame work in patched methods; use events or one-time init in `OnLoad`.
4. Test with DEV.SDK deploy before packing for RDYTU.
5. Do not patch `Postfix` with `__result` on methods Cecil cannot rewrite — use `Prefix` only (see UniversalLoadout).
